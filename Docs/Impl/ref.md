# Unityにおける次世代ベイクエンジンの技術設計
## Substance Painter級の高速処理を実現するためのアルゴリズムと最適化実装

現代のデジタルコンテンツ制作において、アンビエントオクルージョン（AO）マップや曲率マップのベイク処理は、アセットの視覚的な質を決定づける不可欠なプロセスである。しかし、従来のベイク手法は膨大な計算リソースを必要とし、アーティストのイテレーションを妨げる要因となってきた。

Substance Painterのような業界標準ツールと同等のパフォーマンスをUnityのエディタ拡張で実現するためには、単なるアルゴリズムの実装にとどまらず、GPUアーキテクチャに最適化されたデータ構造、専用ハードウェアの戦略的活用、および数学的近似手法を組み合わせた多角的な設計が求められる。本報告書では、高性能なベイカーを構築するための技術的要件、具体的なアルゴリズム、および最適化戦略について、実装レベルの洞察を詳説する。

---

## 1. 空間データ構造の極致：交差判定の高速化戦略
レイトレーシングに基づくベイク処理において、最も計算コストが高いのはレイとメッシュの交差判定である。数百万ポリゴンを超えるハイポリゴンメッシュに対して、空間を効率的に分割して探索範囲を限定する**加速構造（Acceleration Structures）**の選択がシステムの根幹を成す。

### 均一グリッドと非条件分岐型3D-DDAの活用
均一グリッド（Uniform Grid）は、空間を一定サイズの立方体（ボクセル）に分割する構造である。
*   **特性**: ローポリメッシュとハイポリメッシュが空間的に近い場合、高い局所性（Locality）を発揮する。
*   **3D-DDAの最適化**: 通常の3D-DDAは条件分岐（if文）を多く必要とし、GPU上での「分岐ダイバージェンス」を招く。
*   **非条件分岐型3D-DDA (3D-DDA-nc)**: 算術演算、比較演算、論理演算（minやstep関数）を組み合わせることでif文を排除する。
*   **効果**: 実験データによれば、GPU上でのスループットは従来の**1.4倍から2.6倍**に向上する。

### BVHの動的構築とスタックレス探索
境界ボリューム階層（BVH）は、幾何学的な密度に応じて適応的にツリーを構築するため、メモリ効率と探索速度のバランスに優れている。
*   **表面積ヒューリスティック (SAH)**: レイが子ノードにヒットする確率を表面積比から計算し、探索コストを最小化する分割点を決定する。
*   **ビニング手法**: 各軸を「ビン」に分割して最適解を近似することで、$O(n \log n)$ の計算量で高品質なツリーを構築する。
*   **スタックレス探索**: スレッドあたりのレジスタ消費を抑え、占有率（Occupancy）を高めるための手法。

| 手法名 | メカニズム | メリット | デメリット |
| :--- | :--- | :--- | :--- |
| **親ポインタ**<br>(Parent Pointers) | 各ノードに親へのポインタを持たせ、探索終了時に遡上する。 | スタックが完全に不要になる。 | メモリ消費が微増し、探索順序の制御が複雑になる。 |
| **エスケープインデックス**<br>(Skip-Node Pointers) | 交差に失敗した際、次にスキップすべきノードのインデックスを保持する。 | 実装が単純で、線形的な探索が可能。 | 常に固定の順序で探索するため、動的な最適化が困難。 |
| **リスタートトレイル**<br>(Restart Trail) | ビットマスク等を用いて探索済みの階層を記録し、ルートから再開する。 | 動的な探索順序をサポートしつつメモリを節約。 | 同一ノードを複数回訪問するため、計算効率が落ちる場合がある。 |

## 2. SDF（符号付き距離場）とコーントレーシングの統合
交差判定の負荷をさらに削減するアプローチとして、メッシュをSDF（Signed Distance Field）として事前計算し、コーントレーシングを活用する手法がある。

### SDFの高速生成アルゴリズム
効率的な生成手法として以下のものが挙げられる：
*   **特性走査変換 (CSC)**: メッシュの面、辺、頂点から特定のボリュームを押し出し、その範囲内のボクセルに対して最短距離を並列計算する。
*   **テトラヘドラルシェル（四面体殻）を用いたスキャンコンバージョン**: 各三角形に対して5つの四面体を生成し、ボロノイ領域をカバーすることで、不完全なメッシュ（非多様体や穴あき）に対しても頑健な処理を可能にする。

### コーントレーシングによるAO近似
SDFを用いることで、AO計算を「複数の太いレイ（コーン）」による探索に置き換えられる。
*   **仕組み**: ステップごとにコーンの半径 $r$ を拡大させながらSDF値をサンプリングし、遮蔽率を累積する。
*   **効率**: 半球をカバーするために5～6本のコーンをトレースするだけで、数百本のレイを飛ばした場合に近い滑らかなコンタクトシャドウを再現できる。

## 3. 離散微分幾何学に基づく曲率計算の最適化
曲率マップは「エッジの摩耗」や「溝の汚れ」を表現するためのマスクとして重要である。メッシュの幾何学的特性を直接解析する離散微分幾何学の手法を用いることで、効率と精度を両立できる。

### ラプラス・ベルトラミ演算子の離散化
曲率計算の核心は、多様体上のラプラス演算子をメッシュに適用した「ラプラス・ベルトラミ演算子」である。
*   **コタンジェント形式 (Cotangent Formula)**: 隣接三角形の角 $\alpha_{ij}$ と $\beta_{ij}$ の余接を用いて重みを計算する。
$$\Delta f(x_i) \approx \frac{1}{2A_{Mixed}} \sum_{j \in N_1(i)} (\cot \alpha_{ij} + \cot \beta_{ij})(f(x_i) - f(x_j))$$
*   **$A_{Mixed}$ (混合面積)**: ボロノイ領域と重心領域を組み合わせ、鋭角や鈍角が混在するメッシュでも数値的安定性を確保する。

### 曲率の種類と実用的応用
| 曲率の種類 | 数学的定義 | ベイクにおける視覚的効果 |
| :--- | :--- | :--- |
| **平均曲率** (Mean Curvature) | $\kappa_H = (\kappa_1 + \kappa_2) / 2$ | 表面の全体的な凹凸を表現。エッジ摩耗のマスクに最適。 |
| **ガウス曲率** (Gaussian Curvature) | $\kappa_G = \kappa_1 \cdot \kappa_2$ | 鞍点や球状突起の検出。特定の特徴抽出に利用。 |
| **最大・最小主曲率** | $\kappa_1, \kappa_2$ | 異方性のあるエッジや、一方向の曲がりを特定する。 |

Unityの実装では、Compute Shaderを用いて全頂点のコタンジェント重みを並列計算し、テクスチャへ書き込むフローが最速である。

## 4. 最小二乗法による頂点ベイクとインスタンス最適化
量産されるオブジェクトに対しては、テクスチャではなく「頂点への信号ベイク」がメモリ効率の面で有効である。

### 連続最小二乗法（Continuous Least Squares）の数理
頂点での単純サンプリングによるアーティファクトを防ぐため、$L_2$ 誤差を最小化する問題を解くことで滑らかな結果を得る。
*   **線形方程式**: $Ax = b$ に帰着。
    *   **質量行列 $A$**: メッシュの接続関係と面積から算出される対称正定値行列。
    *   **ターゲットベクトル $b$**: 入力信号と基底関数の積の積分値（モンテカルロ・サンプリングで近似）。
*   **解法**: PARDISO等のソルバ、または反復法（共役勾配法）を用いて解く。

### ローカルAOとグローバルAOの線形結合
インスタンス化されたオブジェクト向けに、AOを二つの成分に分解する。
1.  **ローカルAO（固有遮蔽）**: オブジェクト自体の形状による遮蔽。全インスタンスで共有。
2.  **グローバルAO（環境遮蔽）**: シーン配置による遮蔽。球体（Approximating Spheres）等で近似し、実行時にローカルAOと組み合わせる。

## 5. デノイジング技術：極小サンプルからの再構築
計算の「質」ではなく「量」を削減し、少数のレイで計算した結果を協力なデノイズアルゴリズムで補完するのが現代の標準である。

### SVGF（時空間分散ガイドフィルタ）の構造
時空間のデータを統合して高品質な画像を再構成する。
1.  **時空間再投影**: 以前のフレームのデータを現在の結果と合成。法線や深度勾配（$\nabla z$）を用いてゴーストを抑制。
2.  **分散の推定**: 各ピクセル周囲の色のばらつきからノイズを測定。
3.  **エッジ回避型À-trousウェーブレットフィルタ**: サンプリング距離を指数関数的に広げながら（1, 2, 4, 8, 16...）、深度・法線・輝度を考慮して重み付け平滑化を行う。

| フィルタの種類 | 特徴 | ベイクへの適性 |
| :--- | :--- | :--- |
| **ガウスフィルタ** | 単純な平滑化。 | エッジがぼやけるため不適。 |
| **バイラテラルフィルタ** | 輝度差を考慮。 | ノイズが多いと効果が限定的。 |
| **À-trousフィルタ** | 階層的・広範囲の平滑化。 | AOの滑らかなグラデーションに最適。 |
| **クロスバイラテラル** | 外部バッファ（法線・深度）を参照。 | シャープなエッジ維持に不可欠。 |

### ブルーノイズによる誤差分布の最適化
ホワイトノイズの代わりに**ブルーノイズ**を用いることで、エラーを高周波側に分布させ、平滑化フィルタでの除去を容易にする。サンプル数を4～8程度に抑えても、高品質な出力を得ることが可能になる。

## 6. ハードウェア・アクセラレーションとAPIの最適化
GPUのハードウェア特性、特にDirectX Raytracing（DXR）を最大限に活用する。

### Unity DXR APIとインライン・レイトレーシング
*   **RayTracingAccelerationStructure**: RTコアを搭載したハードウェア向けAPI。
*   **TraceRayInline**: Compute Shader内でハードウェア加速された交差判定を実行。
*   **主要プログラム**: Ray Generation、Closest Hit、Miss シェーダの協調。

### SIMD効率とパケットトレーシング
*   **パケットトレーシング**: 近接するレイを一つのパケット（4x4など）としてグループ化し、一括してBVHテストを行うことでメモリ帯域を節約。
*   **マルチBVH (N-way BVH)**: 1ノードが4つ以上の子を持つBVH。ツリーの深さを抑え、高スループットを実現。

### 永続スレッド（Persistent Threads）パターン
各レイの実行時間が異なることによるアイドル状態（Hostage Warps）を避けるため、スレッド自身がグローバルキューから動的にタスクを取りに行く実装。

## 7. ワークフローとテクスチャ空間レンダリング
最終出力がテクスチャであるため、計算の起点をUV空間に置く設計が実用的である。

### UV空間へのアンラップとラスタライズ
*   **手法**: 頂点シェーダで出力座標をUV座標にリマップし、ピクセルシェーダでワールド座標や法線を復元してレイトレーシングを開始する。
*   **利点**: GPUのラスタライザを「どの三角形がどのテクセルをカバーしているか」の判定に流用できるため、複雑なハッシュテーブル等が不要になる。

### 境界部分の処理：ダイレーション（Dilation）
UVアイランドの境界で発生するアーティファクトを防ぐため、アイランド内部の色を外側に転送する「ダイレーション」パスを実行する。これによりミップマップ生成時のブリーディングを防ぐ。

---

## 8. パフォーマンスと品質の最適バランス
*   **AAとセカンダリレイの分配**: 無暗にアンチエイリアス（AA）を上げるのではなく、AAは2x2程度に抑え、AO用の二次レイ（セカンダリレイ）の数を増やす方が時間効率が良い。
*   **幾何形状の事前処理**: 事前に三角形化（Triangulate）を済ませておく。
*   **早期打ち切り**: 影響を及ぼす最大距離（Max Distance）を適切に制限し、BVH探索を高速化する。

---

## 結論と推奨される実装ロードマップ
UnityでSubstance Painterに匹敵するベイクエンジンを構築するための階層的アプローチ：
1.  **ハードウェアBVH (DXR API)** を第一選択とする。
2.  **ブルーノイズサンプリング + À-trousフィルタ** により、レイの総数を劇的に削減。
3.  **離散ラプラス・ベルトラミ演算子** をCompute Shaderで実装し、曲率を瞬時に算出。
4.  **テクスチャ空間ラスタライズ** を核に据え、リアルタイムに近い制作体験を実現。

これらの技術要素を緻密に組み合わせることで、Unityのエディタ拡張は最高峰のテクスチャ制作環境へと昇華される。
